# 发货计划统计

自动处理销售单数据，生成发货计划透视表，帮助快速了解各渠道的发货状态和金额分布。

## 什么时候使用

- 用户说"统计发货计划"、"发货计划统计"
- 需要分析订单的客审状态和发货时间类型
- 需要按销售渠道统计发货金额
- 需要生成发货状态透视表

## 输入要求

Excel文件，包含"销售单"子表，必须包含以下列：
- 标记（用于判断客审状态）
- 订单-发货通知状态（用于判断发货时间类型）
- 订单-人工承诺发货时间
- 销售渠道
- 应收合计

## 处理逻辑

### 1. 插入"客审？"列
判断"标记"列是否含"已客审"或"已货审"
- 含"已客审"或"已货审" → "已客审"
- 否则 → "未客审"

### 2. 插入"发货时间类型"列
根据"订单-发货通知状态"判断：
- "加急发货"或"指定时间发货" → "指定时间"
- 含"有货就发"或"等通知" → "等通知"
- 空白 → 空白

### 3. 修改时间格式
将"订单-人工承诺发货时间"格式化为"YYYY年MM月"

### 4. 插入"发货订单类型"列
合并"客审？"和"发货时间类型"列，中间用"-"连接

### 5. 插入"发货？"列
判断最终发货状态：
- 含"未客审" → "未客审"
- 含"已客审-指定时间" → 对应月份
- 含"已客审-等通知" → "等通知"

### 6. 创建透视表
- 行：销售渠道（剔除空白）
- 列：发货状态（月份从小到大，然后是"等通知"、"未客审"、"合计"）
- 值：应收合计求和
- 格式：万元

## 输出文件

- Excel文件：`发货计划统计结果{日期}.xlsx`（包含处理后数据和透视表）
- 透视表图片：`发货计划透视表{日期}.png`

## 执行方式

脚本位于 `scripts/delivery_plan_stats.py`，可通过以下方式执行：

```python
import sys
sys.path.insert(0, "SKILL_DIR/scripts")
from delivery_plan_stats import main

# 设置输入文件路径
import delivery_plan_stats
delivery_plan_stats.INPUT_FILE = "你的文件路径.xlsx"
delivery_plan_stats.OUTPUT_DIR = "输出目录/"

# 执行
main()
```

## 注意事项

1. 确保输入文件包含"销售单"子表
2. 列名必须与要求一致
3. 输出文件自动添加日期后缀（格式：YYYYMMDD）
4. 透视表会自动处理中文显示
