# 未发货订单统计

自动处理销售单数据，生成发货计划透视表，帮助快速了解各渠道的发货状态和金额分布。

## 什么时候使用

- 用户说"统计发货计划"、"发货计划统计"、"未发货订单统计"
- 需要分析订单的客审状态和发货时间类型
- 需要按销售渠道统计发货金额
- 需要生成发货状态透视表

## 输入要求

Excel文件，第一个sheet为原始数据，必须包含以下列：
- 标记（用于判断客审状态）
- 订单-发货通知状态（用于判断发货时间类型）
- 订单-人工承诺发货时间
- 销售渠道
- 应收合计

## 处理逻辑

### 步骤1: 添加"客审状态"列
- "标记"列含有"客审"或"货审" → "已客审"
- 否则 → "未客审"

### 步骤2: 添加"发货时间类型"列
- "订单-发货通知状态"为"加急发货"或"指定时间发货" → "指定时间"
- 含有"有货就发"或"等通知" → "等通知"
- 空白 → 空白

### 步骤3: 格式化日期
将"订单-人工承诺发货时间"格式化为"YYYY年MM月"

### 步骤4: 添加"状态合并"列
合并"客审状态"和"发货时间类型"，中间用"-"连接

### 步骤5: 添加"统计项目"列
- 含"未客审"字样 → "未客审"
- 含"已客审-指定时间" → 对应月份（YYYY年MM月）
- 含"已客审-等通知" → "等通知"
- "已客审-"（发货时间类型为空）→ "未客审"

### 步骤6: 创建透视表
- 行：销售渠道（剔除空白）
- 列：统计项目（自然顺序）
- 值：应收合计求和

### 步骤7: 创建统计表
- 从透视表取数据，调整列排序：销售渠道 | 日期从小到大 | 等通知 | 未客审 | 合计
- 金额转为万元，显示为"XX万"
- 添加标题行"未发货订单统计 XX年XX月XX日"
- 表格美化

### 步骤8: 生成截图
- 行高紧凑，销售渠道列加宽，其他列缩窄
- 数值带"万"单位

## 输出文件

- Excel文件：`发货计划统计结果{日期}.xlsx`（含3个sheet：处理后数据、透视表、统计表）
- 截图：`发货计划透视表{日期}.png`

## 执行方式

脚本位于 `scripts/delivery_plan_stats.py`，可通过以下方式执行：

```python
import sys
sys.path.insert(0, "SKILL_DIR/scripts")
from delivery_plan_stats import main

# 设置输入文件路径
import delivery_plan_stats
delivery_plan_stats.INPUT_FILE = "你的文件路径.xlsx"
delivery_plan_stats.OUTPUT_DIR = "输出目录/"

# 执行
main()
```

或命令行：
```bash
python3 scripts/delivery_plan_stats.py
```

## 注意事项

1. 输入文件第一个sheet必须是原始数据（不再要求sheet名必须为"销售单"）
2. 客审状态判断：标记列含有"客审"或"货审"即可（不要求"已客审"/"已货审"）
3. 输出文件自动添加日期后缀（格式：YYYYMMDD）
4. 金额单位：万元
5. 列顺序固定：日期从小到大 → 等通知 → 未客审 → 合计
